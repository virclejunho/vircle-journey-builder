BUCKLE ATOMIC FLOW BUILDER

선택만 하면 완성되는
버클 캠페인 화면 플로우

온라인/오프라인 · 이벤트 유형 · 조건만 고르면, 버클의 검증된 원자 기능(Atomic Action)을 조합해 고객 여정 화면 플로우를 자동으로 만들고, 피그마 컴포넌트 매핑 · 일정 · 할 일까지 한 번에 뽑아 줍니다.

핵심: AI 캠페인 기획툴이 아니라 피그마 반복노동 제거툴

WHY · 왜 만드나

AI가 기획을 대신하는 게 아니라,
피그마 반복노동을 없앱니다

진짜 힘든 지점은 캠페인 기획이 아니라, 피그마에서 실제 화면을 가져오고 · 화살표로 연결하고 · 여정도를 그리는 반복 작업입니다. 그래서 빌더는 복잡한 기획을 대신하지 않고, 버클이 이미 운영 중인 화면을 원자 기능 단위로 조립해 줍니다.

지금 — 피그마 수작업

  • ✕실제 화면 따오고 화살표 연결 반복
  • ✕여정도·연결선 매번 처음부터
  • ✕분기·상태·운영조건은 따로 챙김
  • ✕화면 역할이 자산으로 안 남음

HOW · 어떻게 동작하나

3가지만 고르면 → 화면 플로우

1

진입 · 이벤트 선택

온라인/오프라인으로 진입점(QR·CRM·라운지)을 정하고, 럭키드로우·RSVP·예약·포토카드·쿠폰·체크인·설문 중 만들 이벤트를 고릅니다.

2

조건 · 일정 선택

회원가입·구매자 한정·선착순·공유·체크인 같은 조건을 켜고, 시작·종료일을 넣으면 준비·발송 일정까지 자동으로 잡힙니다.

3

화면 플로우 자동 생성

원자 기능을 조합한 화면 플로우 · 화면 리스트 · Figma 컴포넌트 · 배치 JSON · 일정 & 할 일이 한 번에. 화면별 이미지·일정은 여기서 바로 편집합니다.

ATOMIC · 원자 기능 조합

10개 원자 기능을 블록처럼

버클 고객 여정은 아래 원자 기능의 조합으로 구성됩니다. 이벤트를 고르면 그에 맞는 원자 기능 체인이 자동으로 조립됩니다.

⬇

Figma 배치용 JSON

화면 순서·좌표·연결선이 담긴 JSON을 복사/다운로드. 플러그인·MCP가 그대로 배치합니다.

🖼️

화면별 이미지 업로드

각 화면 카드의 ＋ 사진 버튼으로 제품 이미지를 화면마다 따로 올립니다.

🗓️

편집형 일정 & 할 일

날짜·담당이 자동 배치된 준비 체크리스트. 직접 고치고 추가해 고객사와 공유합니다.

🔀

분기 · 상태값 자동

당첨/미당첨·회원/비회원·예약 마감 같은 분기와 상태를 성공 케이스와 함께 그립니다.

✅

운영 체크리스트

개인정보·발송채널·쿠폰 수량·당첨 방식 등 화면 외 운영 조건을 자동으로 정리합니다.
